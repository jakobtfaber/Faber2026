# Domain Docs

This repo has two linked contexts.

## Before exploring

- Read root `CONTEXT.md` for manuscript claims, trust state, and project
  language.
- Read `pipeline/CONTEXT.md` for fitting, data-product, or pipeline work.
- Read relevant decision records under root `docs/adr/` and pipeline-local
  decision directories when present.
- If a file is absent, proceed silently.

## Use the glossaries

Use each context's defined terms. Surface contradictions instead of silently
overriding them. Use `/domain-modeling` when a term itself must be resolved.

This established two-context layout does not require a root `CONTEXT-MAP.md`.
