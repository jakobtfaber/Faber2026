## Question / Scope

Map the repository files, environments, processes, launch agents, ports, and publication dependencies for Jupyter, the MkDocs analysis site, and Manuscript pulse / Running Notes. Confirm local stopped or running state without starting, stopping, restarting, deploying, or probing the public site.

## Findings

### Runtime state

Read-only probes on 2026-07-20 found:

- **Jupyter: stopped.** No Jupyter process matched the full process table. No tracked Jupyter configuration or notebook file exists in this checkout.
- **MkDocs: stopped.** No MkDocs process matched. Nothing listened on TCP port 8000. The repository exposes only a manual `cd docs/analysis && mkdocs serve` instruction (`docs/analysis/index.md:13`).
- **Running Notes: running, not stopped.** Launch agent `com.jakobfaber.faber2026-notes` was loaded in `gui/501`, state `running`, PID 31117. That process listened only on `127.0.0.1:18765`; port 8000 remained closed. The installed property list was byte-identical to the tracked copy. The tracked copy uses `/usr/bin/python3`, the repository script, `--port 18765`, `RunAtLoad`, and `KeepAlive` (`docs/rse/ops/running-notes/com.jakobfaber.faber2026-notes.plist:5`, `:7`, `:15`, `:17`).
- **Tunnel dependency: running locally.** Launch agent `com.cloudflare.cloudflared.orchestrator` had PID 29543. Local config and its Maistro template both route `faber2026.jakobtfaber.com` to `localhost:18765`. Public reachability, Domain Name System, Cloudflare Access policy, and end-to-end authentication were deliberately **unverified**.

### Ownership: MkDocs and Jupyter

- MkDocs source authority is `docs/analysis/mkdocs.yml` plus Markdown and Scalable Vector Graphics assets under `docs/analysis/`. The configuration sets `docs_dir: .`, Material theme, navigation, Markdown extensions, and a MathJax script fetched from jsDelivr (`docs/analysis/mkdocs.yml:1`, `:6`, `:8`, `:21`, `:33`). The overview says producers remain under `analysis/` and `pipeline/analysis/`; the documentation tree owns readable prose and plot panels (`docs/analysis/index.md:1`).
- The default generated MkDocs output is `docs/analysis/site/` (inferred from MkDocs defaults because `site_dir` is unset). It was absent and is not tracked. Regeneration depends on an external `mkdocs` executable with the Material theme and listed extensions. No repository environment or lock file pins those documentation dependencies. The MathJax display also depends on the external jsDelivr URL (`docs/analysis/mkdocs.yml:33`).
- There is no repository-owned Jupyter surface: no tracked notebook, Jupyter configuration, kernel specification, launch agent, environment declaration, generated export, or restart command was found. Any earlier Jupyter server was therefore an ad hoc local process, not a reproducible repository service. Its environment and generated assets are **unverified / unowned by this repository**.

### Ownership: Running Notes

- Service code authority is `scripts/running_notes.py`; it is Python-standard-library-only and binds a `ThreadingHTTPServer` to loopback. The script defines the operations directory, inbox, dispositions, live status, static HTML, pulse JSON, board, map, default port 18765, Claude model, and effort (`scripts/running_notes.py:1`, `:31`, `:42`).
- Authored UI authority is tracked `docs/rse/ops/running-notes/index.html`. The authored status-map authority is tracked `pulse.json`, edited directly and served as `/api/pulse` (`docs/rse/ops/running-notes/README.md:46`, `scripts/running_notes.py:410`). These are served directly; there is no build or generated-site deployment step.
- Runtime-generated state is split deliberately. `inbox/notes.jsonl`, per-note Claude logs, and `status.json` are local mutable products and gitignored (`docs/rse/ops/running-notes/README.md:74`, `.gitignore:63`). `dispositions/<id>.json` is generated by Claude but may be selected for tracking (`docs/rse/ops/running-notes/README.md:51`).
- Submit depends on the Claude Code command-line program, subscription authentication, repository working directory, `BOARD.md`, the ApJ wayfinder map, verification vocabulary, and `CONTEXT.md`. The command is fixed to Claude Opus 4.8 at xhigh effort with standard input closed (`scripts/running_notes.py:2`; `docs/rse/ops/running-notes/README.md:51`). Failure writes a disposition and preserves the note rather than making the HTTP origin unavailable (`scripts/running_notes.py:245`, `:268`).
- Operator entry points are direct Python or `make notes-serve` / `make notes`; both call the same script (`docs/rse/ops/running-notes/README.md:21`, `Makefile:49`). Persistent restart ownership is the installed user launch agent; its logged output goes to `~/Library/Logs/faber2026-notes.{stdout,stderr}.log` (`docs/rse/ops/running-notes/com.jakobfaber.faber2026-notes.plist:21`).

### Restart and publication dependency map

1. **MkDocs local preview:** documentation source + external MkDocs/Material environment -> `mkdocs serve` from `docs/analysis/` -> default loopback port 8000. No persistent agent or publication target is declared. Currently stopped.
2. **Running Notes local origin:** tracked script + tracked `index.html` / `pulse.json` + writable ignored state directories -> `/usr/bin/python3` -> user launch agent -> `127.0.0.1:18765`. Currently running.
3. **Running Notes submission:** local origin -> Claude Code executable + active subscription authentication -> repository authority files -> ignored inbox/log/status plus optional tracked disposition. Origin health does not prove submission authentication.
4. **Public publication surface:** running local origin -> `com.cloudflare.cloudflared.orchestrator` -> `~/.cloudflared/config.yml` ingress -> Cloudflare tunnel, Domain Name System, and Access policy -> public hostname. Local origin and tunnel processes were observed; the outward chain remains unverified.

No Jupyter restart or deployment chain is repository-defined. No service was changed during this inventory.

## Synthesis

Port 8000 closure proves the temporary MkDocs/Jupyter-style preview surface is down, not that all manuscript operational surfaces are down. Running Notes is a separate, persistent loopback origin on port 18765 and was live under launchd. The repository clearly owns its code, static UI, editable pulse, launch-agent template, and state-file policy. It does not own a reproducible Jupyter environment, pin the MkDocs documentation environment, or prove the public Cloudflare edge from local configuration alone.
