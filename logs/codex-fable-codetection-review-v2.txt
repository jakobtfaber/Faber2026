The windowing and marginal fixes landed, but the NaN/hatch implementation broke waterfall rendering on every burst it touched. **6/12 are manuscript-ready or close** (casey, chromatica, freya, hamilton, zach, and marginally isha); **6/12 now have solid-black, unreadable waterfalls** (johndoeII, mahi, oran, phineas, whitney, wilhelm). ## 2. Must-fix scorecard

1. **TOA alignment — resolved at code level.** Offset semantics documented, residual separation intentional. Caption obligation still open: whitney's ~27 ms separation needs the convention stated or it reads as a bug.
2. **3 ms padding — resolved.** mahi −11.7..+17.7 (was −9..+37), oran −6.3..+12.3 (was ..+29); trailing dead space gone, pads check out visually.
3. **Unequal snippets — mechanism right, execution broken.** The two-tone texture is gone (phineas, whitney correctly hatched), but the NaN padding introduced regressions A and B below.
4. **Faint CHIME counterparts — partial.** Per-band normalized profiles/spectra now show isha/oran/mahi CHIME in the *marginals* (real improvement — isha's profile finally demonstrates co-detection). Waterfalls still show nothing, and mahi's is now unreadable. The requested waterfall downsampling wasn't done.

Per-band spectrum marginal (nice-to-have 5): done on all 12. Still open: print-contrast clip (chromatica/hamilton DSA faint), grey DSA masked channels (hamilton ~1380 MHz and phineas ~1410 still white).

## 3. New issues introduced

- **Regression A — NaN poisons the color scale.** Every figure with a hatched span renders its *valid* data as a flat black box at the colormap floor: johndoeII DSA, mahi DSA+CHIME, oran DSA, phineas DSA, whitney DSA+CHIME, wilhelm DSA+CHIME. Telling detail: whitney's DSA band is solid black across the full window even though only CHIME has NaNs — scaling is likely computed on the joint composite, or with non-NaN-aware percentile/vmin-vmax. Needs nanpercentile per band + `cmap.set_bad` for NaN pixels.
- **Regression B — DSA valid-data extent collapsed to slivers.** johndoeII (~3.5 ms), mahi (~1 ms), oran (~1.5 ms), wilhelm (~1.5 ms) show DSA data only in a tiny box around the peak, rest hatched "no data." The prior figures showed full-window DSA noise for these bursts with no two-tone artifact — that time was real data, now mislabeled missing. Smells like a frame mismatch between the CHIME shift/midpoint centering applied to the data vs the coverage mask used for hatching.
- Minor: CHIME masked-channel grey strips draw across hatched no-data spans (whitney, wilhelm) — cosmetic layering.
- Good new behavior worth keeping: profile traces truncate honestly to each instrument's valid span.

## 4. Prioritized recommendations

1. Fix NaN-aware per-band color scaling (regression A) — single fix, unblocks 6 figures.
2. Audit coverage-mask vs data time-frame (regression B) — DSA snippets are not 1 ms.
3. Re-render, then revisit faint CHIME waterfall visibility (isha, oran, mahi) via downsampling.
4. Caption: state alignment reference and that residual separation is genuine.
5. Carry-overs: grey DSA masks, percentile clip for print.
